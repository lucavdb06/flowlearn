# architecture des documents

### folder :

projet : contient une arborecne de dossier qui représente le flux pour crée un documents les folder 2 et 3 sont une boucle tant que la relecture n'ai pas validé

```txt
├── 1-document
│   ├── projet : tout les document lié au projet
│   │   ├── 1-a-faire  // la structure du documment n'ai pas encore définis
│   │   ├── 2-en-cours  // le travail est en cours (un todo et définie dans le document pour définir les taches a faire)
│   │   ├── 3-relecture  // reunion pour faire relire au groupe
│   │   └── 4-terminer // document a ne plus toucher
│   └── techno.md
└── README.md
```
